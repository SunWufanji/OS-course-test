## 双模架构

### 沙盒模式（Live System）
- 用户在桌面双击图标启动应用
- 后端 ProcessManager 创建进程，分配 CPU/内存资源
- 每秒定时更新资源使用（抖动模拟）
- 前端任务管理器实时显示进程状态和资源占用

### 实验室模式（Lab Mode）
- 用户在实验室中配置进程参数和算法
- 后端一次性计算完整甘特图时间线
- 前端通过动画播放甘特图（扫描线效果）
- 离线数学模拟，不影响沙盒模式

## 后端分层

| 层 | 包 | 职责 |
|---|---|------|
| Controller | `controller/` | 接收 HTTP 请求，返回 JSON |
| Service | `service/` | 核心业务逻辑，协调各组件 |
| Scheduler | `scheduler/` | 调度算法实现（策略模式） |
| Hardware | `hardware/` | CPU/内存/IO 资源池 |
| Process | `process/` | 沙盒进程管理器 |
| Model | `model/` | 数据模型（PCB、状态枚举等） |
| Repository | `repository/` | Spring Data JPA 数据访问 |

## 数据库设计

### execution_log（执行轨迹表）
记录每个进程的执行详情：等待时间、周转时间、完成时间。

### performance_metrics（性能指标表）
每次模拟的聚合统计：平均周转、吞吐量、CPU 利用率。

### scenario_config（实验场景表）
预设的实验场景，进程配置以 JSON 格式存储。

```

├── Desktop (桌面图标网格)
│   └── DesktopIcon (单个图标)
```

## 关键设计决策

1. **PID 隔离**：沙盒进程从 PID 100 开始，调度算法进程用 PID 1-99，互不干扰
2. **策略模式**：所有调度算法实现同一个 `Scheduler` 接口，运行时可切换
3. **定时抖动**：`@Scheduled(fixedRate=1000)` 每秒更新 CPU/内存使用率
4. **首次适应算法**：内存分配使用 First Fit，自动合并相邻空闲块
