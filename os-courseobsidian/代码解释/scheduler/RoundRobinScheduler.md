# RoundRobinScheduler.java — 时间片轮转

**算法简称：** RR（Round Robin）

---

## 选择逻辑

```java
return readyQueue.get(0);
```

直接取就绪队列的**队首**。配合 `ProcessService` 的时间片管理——时间片到后当前进程移到队尾，`get(0)` 就是下一个。

---

## 特点

| 特性 | 说明 |
|------|------|
| 是否抢占 | **是**（时间片到自动切出） |
| 选择依据 | FIFO 队列顺序 |
| 响应时间 | **最短** — 每个进程轮流获得 CPU |
| 公平性 | 最好 — 所有进程平等 |
| 适用场景 | **分时系统**、交互式系统 |
| 关键参数 | 时间片大小（通过 `POST /api/quantum` 设置） |

---

## 时间片的影响

| 时间片 | 效果 |
|--------|------|
| 太大 | 退化为 FCFS |
| 太小 | 上下文切换开销过大 |

RR 的 `selectNext` 本身只取队首，时间片管理完全在 `ProcessService.tick()` 中实现。
